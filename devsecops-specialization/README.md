# DevSecOps Specialization – Hackers do Bem

> **Specialized level** of the Hackers do Bem program (ESR / RNP).
> 80-hour synchronous specialization focused on integrating security into the Software Development Lifecycle (SDLC), CI/CD pipelines, and DevOps culture.
>
> **Start date:** May 4, 2026 | **Duration:** 7 weeks

---

## Why DevSecOps — and Why It Fits My Background

As a **QA Engineer transitioning to Cybersecurity**, DevSecOps is the most natural specialization I could have chosen. Here's why:

| My QA Experience | DevSecOps Application |
|-----------------|----------------------|
| Writing test cases & automation | Writing security test gates in CI/CD pipelines |
| API & regression testing | SAST/DAST automated security scanning |
| Bug lifecycle & reporting | Vulnerability tracking and remediation workflow |
| Understanding the SDLC deeply | Embedding security at every SDLC phase |
| Working with developers daily | DevSecOps requires dev + sec + ops collaboration |
| Using Git & version control | GitLab pipelines, branching strategies, code review |

> DevSecOps is not just a security discipline — it's a **quality discipline applied to security**. My QA mindset is a direct advantage here.
